- **솔루션 한 줄 요약**

## 4. AI 활용 방식 및 주요 프롬프트

### 어떤 AI 도구를 어떻게 사용했는가

**Claude Sonnet 4 (claude-sonnet-4-20250514)** 를 자동 매매 엔진("Calico")으로 활용했다.

매 5분마다 Bybit에서 상위 30개 코인의 실시간 데이터(현재가, 24h 등락률, 거래량, 펀딩비)를 수집하고, Claude API에 전달해 3~5개의 매매 추천을 JSON 형태로 받아온다. 응답 결과는 즉시 벤치마크 오더로 등록되며, 이후 실시간 모니터링 루프(5초 주기)가 TP/SL 도달 여부를 체크해 포지션을 자동 청산한다.

**개발 과정에서도 AI 활용:** 프로젝트 전체 코드를 Claude Code(Anthropic CLI)를 사용해 개발했다. 전략 로직 구현, 백테스터 엔진, 실시간 모니터링 루프, 프론트엔드 컴포넌트 등 대부분의 코드를 AI와 페어 프로그래밍 방식으로 작성했다.

### 기존 방식 대비 달라진 점

| 구분 | 기존 방식 | Profit Lab |
|------|-----------|------------|
| 전략 검증 | 실제 자금 투입 또는 전문 퀀트 인프라 필요 | 코드 없이 웹 UI에서 백테스트 즉시 실행 |
| AI 매매 | 규칙 기반 알고리즘만 가능 | Claude가 시황을 직접 판단해 매매 결정 |
| 성과 비교 | 개별적으로 기록·관리 | AI 모델별 리더보드로 객관적 비교 |
| 개발 생산성 | 전략 1개 구현에 수 주 소요 | AI 페어 프로그래밍으로 3개 전략 + 인프라 단기 완성 |

### 실제 사용한 주요 프롬프트

**AI 자동 매매 시스템 프롬프트 (backend/engine/ai_trader.py):**

```
시스템: "당신은 암호화폐 단기 트레이딩 추천 전문가입니다."

유저:
현재 시각: {timestamp}
사용 가능 자금: ${balance}

[현재 시장 상황]
코인 | 현재가 | 24h 변동 | 24h 거래량 | 펀딩비
...

위 데이터를 바탕으로 단기 트레이딩 추천 3~5개를 JSON 배열로 응답하세요.
각 항목에는 coin, direction(LONG/SHORT), entry, tp, sl, confidence(high/medium/low), reason 포함.
진입가는 현재가와 다를 수 있습니다(지지/저항 기반).
불필요한 설명 없이 JSON만 응답하세요.
```

```
백테스트, 라이브 벤치마크 모든 TradingView 차트에서 설정한 지표가 그대로 유지되었으면 함.
지표 등의 설정은 백테스트, 라이브 벤치마크에 모두 동일하게 적용해줘.
```


## 5. 주요 기능 설명

### 1) 멀티 전략 백테스트 엔진
3가지 독립적인 알고리즘 전략(RSI Divergence, EMA Trend, BB Squeeze)을 Bybit 선물 실제 데이터(1분봉 기준)로 검증한다. 공통 리스크 필터(변동성 스파이크, RSI 과매도, BTC 급락 가드 등)를 적용하고, 수수료(0.04%) + 슬리피지(0.05%) + 10배 레버리지를 반영한 현실적인 P&L을 계산한다. 결과는 코인별 수익률·승률·MDD로 분석되며 TradingView 차트에 진입/청산 마커로 시각화된다.

### 2) 실시간 AI 자동 매매 및 벤치마크
Claude Sonnet 4가 5분마다 시황을 분석해 자동으로 포지션을 잡고, 실시간 모니터링 루프(5초 주기)가 TP/SL 도달 시 자동 청산한다. Dual TP 구조(TP1 도달 시 50% 청산 + SL 손익분기점 이동, TP2 나머지 청산)와 다운타임 복구(서버 재시작 시 1분봉 재생으로 누락된 이벤트 복원)를 지원한다.

### 3) AI 모델 성과 리더보드
AI가 실행한 매매 이력을 모델별로 집계해 승률, Profit Factor, 최대 낙폭(MDD), 누적 손익을 비교하는 리더보드를 제공한다. 모델별 상세 페이지에서 전체 주문 이력, TradingView 실시간 차트, 개별 주문 클릭 시 해당 시점으로 차트 이동 기능을 제공한다.

## 7. 아쉬운 점 / 개선 여지

- **실제 주문 미연동**: 현재 벤치마크 시스템은 가상 시뮬레이션이다. Bybit 실제 API 주문 연동이 빠져있어, 실전 트레이딩 성과와의 괴리가 있다.

- **AI 매매 전략의 일관성 부족**: Claude의 응답이 매번 다를 수 있어 재현 가능한 전략이라 보기 어렵다. 프롬프트 최적화와 A/B 테스트를 통한 성능 개선 여지가 크다.

- **백테스트 데이터 한계**: 현재 1분봉 데이터를 직접 Bybit에서 수집하는 구조라 대량의 과거 데이터 확보에 시간이 오래 걸린다. 전용 데이터 소스가 필요하다.

- **전략 파라미터 UI**: 현재 전략별 파라미터(RSI 기간, BB 배수 등)가 백엔드 config.py에 고정되어 있다. UI에서 직접 조정하며 최적화하는 기능이 없다.

- **알림 기능 부재**: TP/SL 도달, AI 매매 신호 발생 시 Telegram이나 이메일 알림이 없다.


## 8. 기대 효과

- **개인 트레이더의 전략 검증 비용 절감**: 실제 자금 투입 없이 과거 데이터로 전략을 빠르게 검증할 수 있어, 실패 비용을 크게 줄일 수 있다.

- **AI 매매 성과의 객관적 측정**: "AI가 실제로 수익을 내는가"를 감이 아닌 데이터로 검증할 수 있다. 모델별·기간별 성과를 수치로 비교함으로써 AI 활용 가능성을 실증할 수 있다.

- **알고리즘 트레이딩 진입 장벽 완화**: 복잡한 퀀트 인프라 없이 웹 UI만으로 전략 실행·분석이 가능해, 비개발자 트레이더도 데이터 기반 의사결정을 할 수 있다.

- **확장성**: 현재 3개 전략 + 1개 AI 모델 구조이지만, 전략 모듈과 AI 모델을 독립적으로 추가할 수 있는 아키텍처로 설계되어 있어 다양한 전략·모델 비교 플랫폼으로 확장 가능하다.
